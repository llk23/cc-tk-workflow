## 核心概念

### Workflow（工作流定义）
用户通过可视化编辑器创建的 DAG（有向无环图），包含节点和边。

### Node（节点）
工作流中的执行单元。每个节点有输入端口、输出端口和配置。
节点类型：fetch、analyze、generate、transform、condition、output。

### Edge（边）
节点间的连接线，定义数据流向。可附带条件（用于分支判断）。

### Pipeline（管线实例）
工作流的运行时执行实例。包含执行上下文、进度、状态。

### Context Store（上下文存储）
节点间共享的数据层，提供中间数据的存取能力。

## 数据流

```
用户保存工作流 → 存入 SQL Server
用户触发执行 → PipelineEngine 读取定义
              → WorkflowGraph 构建 DAG
              → 拓扑排序确定执行顺序
              → NodeExecutor 依次执行节点
              → ContextStore 管理中间数据
              → 执行完成，结果存入 SQL Server
              → WebSocket 推送进度到前端
```
